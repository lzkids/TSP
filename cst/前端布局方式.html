# 常见的布局概念

# 0.静态布局

网页布局始终按照固定的布局和尺寸来显示，不使用 Flex 之类的弹性盒子方案。

优点：最简单的一种布局，没兼容性的问题

缺点：当屏幕分辨率太低时，会展现出滚动条

场景：传统web网站

# 1.流式布局

流式布局目的是在不同大小的设备上满屏呈现同样网页。

优点：灵活。充分利用浏览器的空间

缺点：宽度安卓屏幕进行适配调整，对大屏幕来说用户体验不好，有些布局元素会显得很长

场景：类似微博等

# 2.自适应布局

自适应布局是为不同的屏幕分辨率分别定义不同的布局。

优点：页面能够兼容不同分辨率的设备

缺点：屏幕太小会发生内容过于拥挤

场景：传统web网站

# 3.响应式布局

响应式布局同样分别为不同的屏幕分辨率定义布局。

优点：一套代码兼容web端、平板、以及手机端网页

缺点：工作量大、UI设计也需要多个版本

场景：同时兼容多种不同设备

# 4.弹性布局（rem/em布局）

用来改变盒状模型的灵活性。

# 5.弹性布局（flex布局）

优点：简便、完整、响应式地实现各种页面布局

缺点：只兼容IE10+的浏览器

场景：三栏式布局、垂直水平居中布局
